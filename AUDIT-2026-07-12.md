## Historique backlog (findings bruts avant priorisation)

### Sécurité / robustesse
- **API-03 (P2 structurel)** : clé secrète Kling signée en JWT HS256 **côté navigateur** → exposée. Vraie faille de conception : à terme, signer via un petit proxy (Cloudflare Worker), pas dans le client.
- **SEC-02 (P2)** : `img-src https:` = canal d'exfiltration ouvert ; `script-src 'unsafe-inline'` supprime la 2e défense. Resserrer la CSP.
- **API-01 (P2)** : erreur au milieu du stream SSE (`data.error`) jamais affichée → l'utilisateur voit un blanc. Surfacer.
- **ARCH-02 (P2)** : `localStorage` plein → exception non rattrapée dans `finish()` → app figée en « génération ». try/catch + purge.

### Backend / API
- API-02 (P2) : médias sauvés avec URLs distantes qui expirent → images mortes après quelques jours. Rapatrier en IndexedDB à la génération.

- API-07 : appels secondaires (titre auto, mémoire, suggestions) sans timeout ni AbortController.
- API-11 : `max_tokens` 8192 (L2458) vs règle « 4096 toujours » → trancher.
- API-08 : modèle SF de repli obsolète `DeepSeek-V2.5` (L3298).
